## Workflow

Load this skill before drafting or editing any text.

### 1. Define the brief
- Ask or infer: purpose, audience, tone, length, and format.
- Examples: "professional email to a client", "technical doc for engineers", "casual social post".

### 2. Outline
- For anything longer than a few paragraphs, produce a brief outline first.
- Get agreement if the user has strong preferences.

### 3. Draft
- Write in the agreed tone and length.
- Use short paragraphs, clear headings, and lists where they help scanning.
- Avoid filler, jargon, and passive voice unless the genre requires them.

### 4. Self-review
- Check: clarity, conciseness, tone consistency, grammar, factual claims.
- Verify that the opening answers the reader's "so what?" question.
- Ensure the conclusion matches the purpose.

### 5. Revise
- Offer 1-2 specific alternatives if the user might want a different angle.
- Apply the user's feedback precisely, without rewriting unrelated sections.

## Output format

1. **Draft** — the text.
2. **Rationale** — why you made the key choices (tone, structure, emphasis).
3. **Suggestions** — optional improvements the user may want.

## Tone guide

- Match the user's language (Chinese / English / etc.).
- When in doubt, default to professional but approachable.
